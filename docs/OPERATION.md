# 操作說明

本文件面向維運或現場管理人員，說明日常操作流程。

## 1. 啟動與停止

### 啟動

本機 client 播放器：

```bat
run.bat
```

中央管理端：

```bat
run-admin.bat
```

手動啟動時請先指定角色：

```powershell
cd D:\MYPRJS\RBS
$env:BANNER_APP_ROLE = "client"
.\.venv\Scripts\python.exe -m uvicorn app.main:app --host 0.0.0.0 --port 8080
```

### 停止

- 在前景 console 執行時，按 `Ctrl + C`
- 若做成背景服務，請透過你的服務管理方式停止

## 2. 日常操作順序

建議照以下順序操作：

1. 登入管理後台
2. 新增影片來源
3. 建立播放清單
4. 建立播放排程
5. 本機模式按「下載目前影片」；client 模式按「同步後台並下載」
6. 到前台確認播放

管理登入說明：

- `client` 前台日常只使用 `/`
- `/admin` 只給管理或維護人員使用
- `/admin` 需輸入管理密碼
- 預設密碼為 `admin`
- 建議首次登入後立即修改

## 3. 前台顯示控制

目前前台支援以下操作：

1. 可依設定決定是否顯示程式名稱
2. 可顯示播放清單自訂抬頭文字
3. 可切換全螢幕
4. 可打開影片清單抽屜並直接跳播指定影片

## 4. 影片管理

後台 `/admin` 可新增影片資料，需填：

- 標題
- 來源 URL
- 是否啟用

也可直接上傳本機影片檔：

- 支援多選
- 上傳後會直接寫入 `storage/media/`
- 系統會自動建立影片資料

說明：

- 新增影片只會建立資料，不會自動立即下載
- 若是直接上傳影片，檔案已在本機，不需要再下載
- URL 來源需按「下載目前影片」或「同步後台並下載」才會抓取本地檔案
- 直接上傳影片已在本機，不需要再下載
- 實際播放只會使用已下載成功且 `enabled=true` 的影片

## 5. 播放清單管理

建立播放清單時需填：

- 清單名稱
- 說明
- 前台抬頭文字
- 在影片選擇區勾選要加入的影片
- 是否啟用

注意：

- 勾選影片後，畫面會依勾選順序建立清單
- 若要調整順序，可用「上移 / 下移」
- 清單存在但若其中影片尚未下載，前台不會播那些未落地的檔案
- `/admin` 已支援新增、編輯、刪除

## 6. 排程管理

排程欄位如下：

- 排程名稱
- 播放清單下拉選單
- 星期按鈕
- 開始時間
- 結束時間
- 每日完整播放次數上限
- 優先序
- 是否啟用

規則：

- `priority` 數字越小越優先
- `play_count=0` 代表無上限
- `play_count>0` 代表該排程在單日內最多允許完整播放整個清單幾次
- 到隔日後，已播次數會自動歸零

## 7. 同步操作

### 下載目前影片

用途：

- 只下載目前資料庫中的影片
- 不會重新匯入清單或排程
- 此按鈕放在 admin 頁面最上方的快速動作區

適用情境：

- 手動新增影片後要下載本地檔
- 不使用中央 manifest，只做本機管理

### 同步後台並下載

用途：

- 由 `central_base_url` 拉取 manifest
- 匯入影片、清單、排程
- 下載影片
- 刪除本機不再被資料庫引用的檔案
- 此按鈕放在 admin 頁面最上方的快速動作區

適用情境：

- 有中央後台或總部統一派送內容

注意：

- 若 `central_base_url` 與 `後台 Manifest URL` 留空，代表目前為本機離線模式
- 本機離線模式下，不需要執行「同步後台並下載」
- 此時請改用本機上傳影片、手動建清單與排程

### 中央端派送狀態

中央端 `/admin` 可查看 client 註冊與回報狀態。若 client 沒出現在清單中，請確認 client 已設定：

- `central_base_url`
- `client_code`
- `client_name`

第一次同步成功後，中央端會產生並保存該 client 的 token。

## 8. 線上更新操作

系統目前提供「版本檢查與下載更新封包」能力，尚未做到自動覆蓋更新。

### 後台設定

- `latest_version`
- `latest_package_url`

### 用腳本下載更新封包

```powershell
.\scripts\update-client.ps1 -ServerBaseUrl http://127.0.0.1:8080
```

下載後封包會放在：

- `storage/updates/`

### 建議流程

1. 先下載新封包
2. 驗證封包內容
3. 停止目前服務
4. 以正式部署流程替換檔案
5. 重新啟動服務

## 9. 常見檢查點

### 系統是否支援離線播放

支援。

說明：

1. 影片播放來源是本機 `storage/media/`
2. 排程與清單來自本機 `storage/banner.db`
3. 因此前台播放本身不依賴即時連線

但以下動作仍需連線：

1. 向後台同步新 manifest
2. 下載新影片
3. 下載新版更新封包

若 `Manifest URL` 留空：

1. 後台會顯示本機離線模式
2. 「同步後台並下載」不適用
3. 前台仍可照常播放本機既有內容

### 前台顯示「目前沒有可播放內容」

請依序檢查：

1. 是否有符合目前時間的啟用排程
2. 排程對應的清單是否啟用
3. 清單中的影片是否啟用
4. 影片是否已成功下載到 `storage/media/`
5. 排程是否已達當日播放次數上限

### 同步沒有下載影片

請檢查：

1. `source_url` 是否可連線
2. `central_base_url` 是否正確；舊版 `upstream_manifest_url` 只作相容欄位
3. 影片是否被標成 `enabled=false`
4. 來源網址是否需要驗證或授權

### 版本顯示有更新但無法下載

請檢查：

1. `latest_package_url` 是否正確
2. 主機是否可連到封包 URL
3. `storage/updates/` 是否有寫入權限

更多錯誤訊息與檢查指令請看 [問題排除](TROUBLESHOOTING.md)。例行備份、還原與升級請看 [維護手冊](MAINTENANCE.md)。
