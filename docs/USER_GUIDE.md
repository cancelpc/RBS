# 使用說明

本文件面向一般使用者或現場操作人員，說明系統如何使用。

## 1. 角色區分

- 前台播放畫面：給現場螢幕使用，只負責播放
- 後台管理畫面：給管理人員使用，用來設定影片、清單、排程與同步
- 中央管理端：給總部或管理單位維護內容與派送
- 本地 client：給現場播放機同步內容並播放本機影片

## 2. 前台播放器

進入：

- [http://127.0.0.1:8080/](http://127.0.0.1:8080/)

功能：

- 自動依照目前時間選擇可播放排程
- 依清單順序連續播放影片
- 每 15 秒重新查詢目前排程
- 若當前排程播完整個清單一次，系統會回報一次已播次數

當沒有內容可播時，畫面會顯示等待訊息。

## 3. 後台管理畫面

進入：

- [http://127.0.0.1:8080/admin](http://127.0.0.1:8080/admin)

畫面改成兩大區：

- 上方「維護首頁」：顯示影片 / 清單 / 排程數量、同步狀態與常用快速動作
- 下方設定卡片：分別維護影片、播放清單、排程、同步與版本

右側固定顯示目前資料摘要，可直接在摘要區編輯或刪除既有資料。

補充：

- `/admin` 需先登入
- 預設管理密碼為 `admin`
- 建議首次登入後立即修改

## 4. 建立第一組播放內容

### 步驟 1：新增影片

在「影片」區輸入：

- 標題
- 來源 URL
- 勾選啟用

按「新增影片」。

若影片檔案就在本機，也可直接使用：

- 「直接上傳影片（可多選）」欄位
- 一次選擇多支影片後按「上傳影片」

上傳完成後，系統會直接把檔案存到本機 `storage/media/`，不需要再透過外部 URL 下載。

### 步驟 2：建立清單

在「播放清單」區輸入：

- 清單名稱
- 說明
- 前台抬頭文字
- 從右側影片選擇器勾選要加入的影片
- 需要時可用「上移 / 下移」調整播放順序

若你只是要把目前全部影片做成一個循環播放清單，也可直接按：

- 「用全部影片建立循環清單與排程」

### 步驟 3：建立排程

在「排程」區輸入：

- 排程名稱
- 播放清單
- 星期
- 開始日期，可留空
- 結束日期，可留空
- 開始時間
- 結束時間
- 每日播放上限
- 優先序

範例：

- 星期：直接點選「每天」或逐一點選週一到週日
- 日期：起訖日都可留空；有設定時包含當日
- 開始時間：`08:00`
- 結束時間：`22:00`
- 每日播放上限：`0`
- 優先序：`100`

### 步驟 4：下載影片

按「下載目前影片」。

### 步驟 5：切到前台檢查

回到 `/`，確認影片是否開始播放。

## 5. 什麼是播放次數

本系統的播放次數不是單支影片次數，而是：

- 完整播放整個清單一次，計 1 次

例如清單內有三支影片：

1. A
2. B
3. C

則播放順序 A → B → C 全部播完，才算 1 次。

## 6. 什麼是優先序

若同一時間有多個排程都符合條件：

- `priority` 數字越小的排程會優先播放

建議做法：

- 重要檔期：設定較小數字，例如 `10`
- 一般輪播：設定較大數字，例如 `100`

## 7. 前台功能

前台使用者可做的事：

- 看目前排程自動輪播
- 視需求切換全螢幕
- 打開影片清單
- 直接點選影片跳播

前台預設不顯示程式名稱；是否顯示由後台設定決定。
前台是否在載入時預設嘗試進入全螢幕，也可由後台設定決定。

## 8. 中央派送使用方式

若有中央後台提供 manifest：

1. 在 client 後台設定 `central_base_url`
2. 設定 `client_code` 與 `client_name`
3. 視需要設定 `client_site_name` 與 `client_group_name`
4. 設定同步秒數
5. 按「同步後台並下載」
6. 之後系統背景執行緒會依設定秒數定期檢查 manifest

若沒有中央後台：

- `central_base_url` 與 `後台 Manifest URL` 留空即可
- 後台會顯示本機離線模式
- 這時候請不要使用「同步後台並下載」
- 直接使用本機上傳影片、建立清單與建立排程即可

## 9. 離線播放與連線更新

本系統支援「離線播放、連線更新」：

- 前台播放時，使用的是本機已下載的影片
- 不需要一邊連後台一邊串流播放
- 只要影片、清單、排程已經存在本機，就算外網中斷，前台仍可繼續播放

需要連線的情境只有：

- 下載新影片
- 更新播放清單與排程
- 下載新版程式封包

因此若中央後台暫時不可用：

- 已下載內容可繼續播放
- 但不會拿到新的影片與新的排程更新

## 10. 一次上傳多支影片後自動循環

如果你的需求是：

- 上傳很多支影片
- 自動一直輪播

建議操作方式：

1. 在 `/admin` 多選上傳影片
2. 按「用全部影片建立循環清單與排程」
3. 系統會自動建立一筆全天排程
4. 若需要，再自行修改排程時段或優先序

這樣前台就會依照清單順序持續循環播放。

## 11. Client 分組與站點

中央端可依派送目標決定哪些 client 會收到內容：

- `global`：所有 client 都收到
- `group`：只有 `client_group_name` 相同的 client 收到
- `site`：只有 `client_site_name` 相同的 client 收到

若 client 同時符合站點與群組，會同時取得全域、群組與站點內容。

## 12. 目前介面限制

目前交付仍是 MVP，但 admin 已調整成較精簡的維護流程：

- `/admin` 可在同頁完成新增、編輯、刪除影片、播放清單與排程
- 播放清單不再需要手動輸入影片 ID
- 排程不再需要手動輸入播放清單 ID
- 版本更新目前仍支援檢查與下載，不支援自動覆蓋升級
- admin session 存在記憶體中，服務重啟後需重新登入
- 中央端正式大型佈署仍建議依 [中央管理端 + 本地 Client 架構草案](CENTRAL_CLIENT_ARCHITECTURE.md) 擴充資料庫、檔案服務與監控
