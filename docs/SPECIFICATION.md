# 需求規格

本文件整理目前系統需求與對應實作狀態。

## 1. 專案目標

建立一套以瀏覽器為前台載體的輪播系統，能在本機穩定播放預先下載好的影片，並可由後台統一派送、排程與更新。

## 2. 功能需求

### FR-01 固定播放選擇的影片

需求：

- 系統需能播放已下載到本機的影片
- 前台不可依賴即時串流

目前實作：

- 已完成
- 影片由 `/media/{file}` 提供，本地播放

### FR-02 支援多組播放清單

需求：

- 系統需可設定多組播放清單
- 每組清單可包含多支影片與順序

目前實作：

- 已完成
- 以 `playlists` 與 `playlist_items` 管理

### FR-03 支援播放時間設定

需求：

- 系統需可設定清單在特定時間播放
- 需可設定星期與時段

目前實作：

- 已完成
- 支援星期、開始時間、結束時間

### FR-04 支援播放次數設定

需求：

- 系統需可設定播放次數限制

目前實作：

- 已完成
- 定義為「單日內完整播放整個清單的次數上限」

### FR-05 定時連到後台取得新影片並下載

需求：

- 系統需可定時連到後台取得新影片資訊
- 需自動下載本地檔案
- 需清除不再使用的影片

目前實作：

- 已完成 MVP
- client 透過 `central_base_url` 拉取中央 manifest，匯入資料並同步檔案
- 保留 `upstream_manifest_url` 相容欄位；設定會與 `central_base_url` 互相補值

### FR-06 可以線上更新版本

需求：

- 系統需具備線上更新能力

目前實作：

- 已完成部分
- 支援版本檢查與更新封包下載
- 尚未做到自動覆蓋檔案與自動重啟

### FR-07 前台以瀏覽器為載體

需求：

- 前台必須以瀏覽器作為播放容器

目前實作：

- 已完成
- `/` 為 HTML5 Video 播放器頁面

## 3. 非功能需求

### NFR-01 可部署於 Windows

- 已符合
- 使用 Python/FastAPI/SQLite，可在 Windows 主機部署

### NFR-02 可離線播放

- 已符合
- 實際播放使用本地影片檔

### NFR-03 簡化維運

- 已提供 `/admin`、管理登入、同步、版本下載、Windows 啟動腳本
- 尚未有完整 RBAC、審計與安裝器

### NFR-04 易於擴充

- 已符合
- API、資料模型與靜態前端分離，後續可擴充正式後台
- 已補上 `central` / `client` 角色、client 註冊、心跳、事件回報與 target scope

## 4. 系統限制

- 目前後台頁面仍偏向 MVP 管理工具
- 已提供管理登入，但 admin session 存於記憶體，僅適合單實例 MVP
- 已提供本機多檔上傳，但未做完整檔案格式與惡意檔掃描
- 無播放紀錄報表
- 無自動覆蓋升級能力
- 中央端目前仍使用 SQLite；正式多 client 大型佈署建議改為 PostgreSQL 或 SQL Server
- 中央端不應承擔大量影片串流，影片檔案服務應與 API 分離

## 5. 後續建議需求

- BR-01 RBAC 權限控管與審計紀錄
- BR-02 影片格式驗證、容量限制與安全掃描
- BR-03 播放紀錄報表與錯誤告警
- BR-04 裝置管理介面與批次設定
- BR-05 正式版自動更新與回滾機制
- BR-06 排程衝突檢查與視覺化日曆
